# 學生詳頁與六次成績拆分設計

## 目標

這一輪要解決兩件事：

1. 把目前只看平均的成績模型，擴充成可直接顯示三次段考與三次平時成績。
2. 新增每位學生的獨立詳頁，並讓其他頁面的學生列都能連到該頁。

本次範圍以現有靜態單頁應用為基礎，不新增後端、不改變資料來源，仍使用 `grades_by_student.json`。

## 使用者需求

- 主管或導師在其他頁面看到學生時，可以直接點入個別學生頁。
- 學生詳頁要能一眼看出同一科的三次段考、三次平時、科目平均與該科 PR。
- 既有的學生搜尋頁不再只是查詢框，而是學生詳頁的入口。
- 其他總覽、班級、科目、關注名單頁仍保留工作台定位，不把所有學生資訊塞回原頁。

## 方案

採用方案 A：學生詳頁中心化。

- 每位學生使用獨立 hash 路由，例如 `#student/LJ14089`
- 其他頁面統一透過學生姓名連結進入詳頁
- 學生詳頁主表採「科目為列、六次成績為欄」

不採用抽屜或彈窗方案，因為那會削弱跨頁連動、返回導覽與網址可分享性。

## 資料模型調整

### 現況

目前 `normalizeGradebook()` 會把每個科目的：

- `regularAssessments`
- `dailyScores`

壓縮成：

- `regularAverage`
- `dailyAverage`
- `overallAverage`

因此前端無法直接知道三次段考與三次平時各自的欄位。

### 新模型

對每筆科目 record 增加固定欄位：

- `exam1`
- `exam2`
- `exam3`
- `daily1`
- `daily2`
- `daily3`

規則：

- 依陣列順序取前三筆，沒有就填 `null`
- 仍保留 `regularScores`、`dailyScores` 與平均欄位，避免破壞現有總覽與排行邏輯

### 學生摘要

在學生摘要中新增可供詳頁直接使用的科目明細，例如：

- `subjectBreakdown[]`
  - `subjectName`
  - `exam1..exam3`
  - `daily1..daily3`
  - `subjectAverage`
  - `subjectPr`
  - `isExamSubject`

這樣學生詳頁不需要再從原始 records 臨時拼接。

## 路由設計

### 新增路由

- `#students`
  - 保留成搜尋入口頁
- `#student/<studentNumber>`
  - 單一學生詳頁

### 路由行為

- 在搜尋頁輸入並選定學生後，直接導向 `#student/<studentNumber>`
- 若路由中的學生不存在，顯示空狀態與返回入口

## 畫面設計

### 1. 學生搜尋頁

保留為入口頁，但功能簡化：

- 搜尋姓名 / 學號 / 班級 / 座號
- 顯示符合的學生結果清單
- 點擊結果進學生詳頁

不在這頁承載完整學生成績表。

### 2. 學生詳頁

頁面分三區。

#### A. 頂部摘要

- 班級
- 座號
- 姓名
- 學號
- 全校排名
- 班排名
- PR
- 整體平均

另保留簡短風險標記：

- 多科不及格
- 整體偏低
- 明顯退步

#### B. 主表：六次成績總表

欄位：

- 科目
- 段考 1
- 段考 2
- 段考 3
- 平時 1
- 平時 2
- 平時 3
- 科目平均
- 科目 PR

呈現規則：

- 空值顯示 `-`
- 低於 60 的欄位做弱提示
- 每列固定寬度與縮排，避免欄位跳動

#### C. 補充區

下方只保留兩個區塊：

- 弱科與追蹤訊號
- 前後學期變化摘要

先不額外加入太多圖表，避免學生頁又變成第二張報表牆。

## 跨頁連動

以下頁面的學生姓名都改成可點連結：

- 全校排名
- 班級分析
- 科目分析
- 關注名單
- 學生搜尋結果

點擊後統一跳轉到 `#student/<studentNumber>`。

為了避免每個表格各寫一套，前端新增共用 helper，例如：

- `studentLink(row)`
- 或在 table render 層支援直接輸出連結節點

## 對現有頁面的影響

### 排名頁

- 不改排序邏輯
- 學生姓名改成連結

### 班級頁

- 班內成績表姓名改成連結

### 科目頁

- 需注意學生列已拆成班級 / 座號 / 姓名
- 只把姓名改成連結，不再增加其他互動

### 關注名單

- 兩張名單的姓名都改成連結

### 學生頁

- 從「單頁查詢結果」改成「搜尋入口頁」
- 真正的資料細節搬到新路由

## 測試

### 資料層

新增測試：

- `normalizeGradebook()` 能正確拆出 `exam1..exam3`、`daily1..daily3`
- 學生摘要能產出 `subjectBreakdown`
- 單科 PR 能正確進入學生詳頁資料模型

### 路由與 UI

最小驗證：

- `node --check app/src/app.js`
- `node --test app/tests/analytics.test.js`
- 本地靜態伺服器確認：
  - `#students`
  - `#student/<studentNumber>`
  - 從排名頁點學生可跳頁

## 不在本輪範圍

- 新增後端或資料庫
- 匯出個別學生報表
- 教師備註、輔導紀錄
- 多學生比較頁
- 歷次學期折線圖的完整視覺化重做

## 實作原則

- 優先沿用現有 `analytics.js` 與 `app.js` 架構，不重做整個 SPA
- 新增資料欄位時不破壞既有排行、風險、班級摘要邏輯
- 學生詳頁聚焦在「看一位學生」，不承擔整批查詢任務
